Foundational 3D Environment Modeling for Proposed Radar Digital Twin Integration

1. Introduction

This report documents the workflow and methodology used to develop four distinct 3D junction models within the Unity engine. This work represents the foundational phase of a Radar Digital Twin project. Because the physical radar hardware has not yet been deployed on-site, the current scope of this project focuses entirely on environment preparation. The objective is to proactively construct accurate virtual replicas of target road intersections, establishing the necessary spatial frameworks so that radar detection data, tracking logic, and sensor telemetry can be seamlessly integrated once the hardware is installed.

2. Methodology & Workflow

The development process follows a systematic pipeline utilizing web-based GIS (Geographic Information System) data and mapping tools to ensure high spatial fidelity.

Stage 2: Spatial Verification & Asset Reference

To build accurate virtual replicas without active hardware feeds, remote sensing assets are used as a reference layout:

  • The extracted coordinates are mapped in Google Maps.
    • Junction location in Google Maps
  • Satellite imagery and street-level views are thoroughly analyzed to evaluate lane counts, intersection boundaries, and surrounding infrastructure geometry.
    • Street view analysis

Stage 3: Virtual Environment Reconstruction in Unity

Using the verified geographic data as a design baseline, the physical intersections are reconstructed inside the Unity engine. Scales, road geometries, and lane markings are modeled to accurately match real-world proportions. To date, four (4) complete 3D junction environments have been successfully modeled.

3. Significance of the Environment-First Approach

Developing the 3D environments prior to hardware installation offers several distinct advantages:

  • Pre-deployment Simulation: Unity models allow developers to simulate radar placement heights, field-of-view ranges, and potential structural blind spots before spending capital on physical installation.
  • Ready-to-Receive Architecture: Having the spatial framework ready means that once real-world radars are installed, data streams can be instantly mapped onto a verified coordinate system.

4. Conclusion & Next Steps

This phase of the project successfully establishes a reliable pipeline for converting JKR asset data into scalable 3D Unity environments, resulting in four finalized junction model

Since the physical radar units are not yet active, the immediate next steps will involve using simulated object data (such as artificial vehicle bounding boxes or waypoint paths) within Unity. This will allow for the testing of the visualization layer and UI elements, ensuring the system is fully operational and waiting for the physical hardware integration.
